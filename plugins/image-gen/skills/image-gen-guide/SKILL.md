# 图片 / 视频生成

使用 `image-gen_generate-image` 和 `image-gen_generate-video` 工具生成媒体内容。

## 非阻塞工作流

生成是异步的。提交后工具立即返回一张卡片，你**不需要等待结果**，也**不需要调用 stage_files**。

1. 按下方规范编写 prompt
2. 调用工具，传入 prompt 和参数
3. **告诉用户图片/视频正在生成，完成后会自动显示在卡片中**
4. **继续对话**，不要等待
5. 当你收到 `<hana-background-result>` 通知时，自然地告知用户结果

## generate-image 参数

- `prompt`（必填）：图片描述，中英文均可
- `count`：并发生成张数（1-4），用户说"多来几张"/"再抽几张"时用
- `image`：参考图路径（图生图、图片编辑、风格迁移时传入）
- `ratio`：长宽比（1:1, 16:9, 9:16, 4:3, 3:4, 3:2, 2:3, 21:9）
- `resolution`：分辨率（2k, 4k）
- `quality`：画质（low, medium, high）
- `provider`：指定 provider（可选，默认自动选择）

## generate-video 参数

- `prompt`（必填）：视频描述
- `image`：参考图路径（图生视频）
- `duration`：时长（秒）
- `ratio`：长宽比
- `provider`：指定 provider（可选）

## Prompt 编写规范

用**简洁连贯的自然语言**描述画面。简洁精确优于堆叠华丽词汇。

### 基本结构

用自然语言写明 **主体 + 行为 + 环境**，有美学需求时补充 **风格、色彩、光影、构图**。

好的 prompt：
> 一个穿着华丽服装的女孩，撑着遮阳伞走在林荫道上，莫奈油画风格。

差的 prompt（关键词堆叠）：
> 一个女孩，撑伞，林荫街道，油画般的细腻笔触。

### 具体化

用户的描述通常简短，你需要补充具体的视觉细节：

- 用户说"猫" → 补充品种、毛色、姿态、表情、场景
  > 一只毛茸茸的灰色长毛猫趴在窗台上晒太阳，眼睛半眯着，窗外是模糊的雨景
- 用户说"风景" → 补充季节、时间、天气、具体场景
  > 深秋的山谷，金色的夕阳从云层间洒下，薄雾从河面升起
- 用户说"海报" → 补充用途、主体、文字内容、排版风格
  > 设计一张音乐节海报，标题为 "ECHO 2026"，主视觉是抽象的声波和星空，暗色背景

### 文字渲染

需要图中出现文字时，把文字内容放在**双引号**里：
> 生成一张咖啡店招牌，上面写着 "Morning Brew"

### 风格控制

有明确风格需求时，使用精准的风格词：
- 绘本、儿童绘本、水彩、油画、赛博朋克、像素风、扁平插画、浮世绘...
- 用户没指定风格时，不要替用户假定风格，让模型自由发挥

### 细节丰富的画面

复杂场景可以通过详细描述精确控制每个元素的位置和状态：
> 一个凌乱的办公桌桌面。桌上有一台开着的笔记本电脑，屏幕显示绿色代码；旁边一个马克杯，杯上写着 "Developer"，杯口冒出热气；一本摊开的书；一支钢笔，笔帽掉在旁边；桌子角落是一小盆多肉植物。阳光从右侧照射，在桌上形成光影。

## 图生图

### 图片编辑

用户想修改图片内容时，传入 `image` 参数，prompt 写**简洁明确的编辑指令**：
- 增加：`给图中女生增加银色耳线和项链`
- 删除：`去掉女生的帽子`
- 替换：`把最大的面包人换成牛角包形象，保持动作和表情不变`
- 修改：`让图中三个机器人变成透明水晶材质`

如果希望其他部分不变，在 prompt 中强调：`保持动作不变`、`不改变其他布局`。

画面复杂时，可提示用户用箭头、线框、涂鸦标注编辑位置，然后在 prompt 中引用：`将红色涂抹位置放入电视`。

### 参考图生图

用户提供参考图想生成新图时，在 prompt 中**指明参考什么 + 描述生成画面**：
- 参考人物形象：`参考图中的人物形象，做一个动漫手办，放在桌上...`
- 参考风格：`参考图标的线性简约风格，设计 9 个应用 icon...`
- 参考款式：`生成四件不同材质和颜色的上衣，衣服款式与图中一致`

### 设计稿转高保真

用户提供线稿、手绘、平面图时：
- 明确主体：`高保真 UI 界面`、`现代简约风格客厅实景图`
- 指出需要保持一致的关键要求：`家具位置与参考图一致`
- 图中有文字说明的：`遵循图中文字内容进行生成`

### 不需要图生图的场景

- 用户发了图片但只是聊天/问图片内容 → 不调用生图工具
- 用户描述全新画面，没提到参考图 → 纯文生图，不传 image

## 注意

- 生成消耗 provider 额度，大批量前建议提醒用户
- 不同 provider 支持的参数不同，工具会自动处理
- 视频生成通常比图片慢（可能几十秒到几分钟），但同样不阻塞
