# 角色提示词去模板感指南

写 `ishiki.md` 和台词范例前先读本页。目标不是让角色故意粗鲁或语病百出，而是把抽象人设改成模型能够稳定执行、用户能够从对话里辨认的行为规则。

## 一、先删掉没有行为含义的形容词

下面这类词单独出现时几乎不能约束对话：

- 温柔、理性、专业、可靠、细腻、毒舌、有分寸
- 外冷内热、嘴硬心软、清醒通透、强大又脆弱
- 永远耐心、总能理解、恰到好处、坚定地陪伴

每保留一个标签，都补一个可观察的表现：

- 不写「他很毒舌」，写「发现论证跳步时先指出断点，不用夸奖垫场；批评只针对文本」
- 不写「她很温柔」，写「用户慌乱时先把下一步缩成一件事，等对方缓过来再解释原因」
- 不写「他外冷内热」，写「平时回答短；用户真遇到麻烦时会把遗漏的步骤补齐，但不邀功」

## 二、给角色留下不那么正确的棱角

角色若永远周到、耐心、正确，就只剩服务口吻。至少写清一个会真实影响对话的局限：

- 他容易在哪件事上失去耐心？
- 他会误判什么？发现误判后怎样收场？
- 他更看重准确、速度、体面还是关系？冲突时舍弃哪一个？
- 他有什么小习惯会让人偶尔觉得麻烦？

缺陷不能伤害用户，也不能成为拒绝基本责任的借口。它应改变语气和选择，而不是降低事实准确性、安全性或任务完成度。

## 三、写语言指纹，不写气氛说明

语言规则要能直接检查：

- 句子通常多长，先下结论还是先问问题
- 常用哪些动词、称呼和连接方式
- 哪些词绝不会说，什么时候会沉默或只回一句
- 高兴、担心、不耐烦时，措辞分别怎样变化

给三句处境不同的台词：日常回应、意见冲突、承认不知道。三句遮掉名字后仍能认出是同一个人，语言指纹才算成立。

避免用旁白替角色宣布情绪，例如「心里泛起暖意」「空气忽然安静」「眼神里多了理解」。让态度落在选词、停顿、追问和行动上。

## 四、警惕整齐得像模板的句子

发现以下现象时重写：

- 连续用相同句式列三点，只是换了名词
- 先否定一个空泛说法，再宣布一个更宏大的说法
- 每段最后都补一句总结、鼓励或人生感悟
- 为了显得深刻，堆比喻、程度词和戏剧化标点
- 用「真正的」「所谓的」「归根结底」代替具体判断
- 无论用户说什么，都先复述、肯定，再进入答案

重写时保留信息量最高的一句，补具体对象、条件或动作。说完即可停，不必替每段安排余韵。

## 五、把原则写成条件与动作

抽象原则必须说明触发条件和做法：

```markdown
## 原则
- 用户只想要结论时，先用两三句回答；需要背景再展开
- 不确定时明确说缺什么证据，并给出验证办法
- 用户的方案有明显漏洞时直接指出，说明后果，同时给一个可行替代
- 用户情绪低落时不急着讲大道理，先确认眼下最难处理的具体事情
```

这类规则比「真诚、善良、专业、有边界」更稳定，因为模型知道何时触发、具体做什么。

## 六、交付前做四个小测试

1. **换名测试**：把角色名换掉，人格是否仍然独特？如果谁都能用，继续具体化。
2. **冲突测试**：用户不同意他时，他的反应是否符合设定，又不妨碍完成任务？
3. **失误测试**：他不知道或做错时会怎样说？没有这一条，角色容易退回万能助手口吻。
4. **删句测试**：删掉一句后角色行为没有变化，这句通常只是装饰。

最终留下的每条设定，都应至少影响一种可见结果：他说什么、怎么说、先做什么、拒绝什么，或在冲突中如何取舍。
